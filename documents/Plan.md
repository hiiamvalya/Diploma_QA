## ПЛАНИРОВАНИЕ

### Исходные данные:
Карты, представленные для тестирования (файл data.json):
1. Банковская карта с номером №4444 4444 4444 4441, статус карты “APPROVED”
2. Банковская карта с номером №4444 4444 4444 4442, статус карты “DECLINED”

### Пункт 1. Перечень автоматизированных сценариев:

#### Позитивные сценарии:

Пользователь совершает покупку тура через кнопку "Купить/ Купить в кредит" с главной страницы сервиса.
1. Успешная покупка при оплате картой с валидным номером 4444 4444 4444 4441 и достаточным количеством средств на счете.
Ожидаемый результат: появилось всплывающее окно "Успешно. Операция одобрена Банком", статус в БД APPROVED.

2. Покупка тура при оплате картой с валидным номером 4444 4444 4444 4442 и недостаточным количеством средств на счете.
Ожидаемый результат: появилось всплывающее окно "Ошибка! Банк отказал в проведении операции", статус в БД DECLINED.

3. Успешная покупка при оплате картой с валидным номером 4444 4444 4444 4441 с достаточным кредитным лимитом.
Ожидаемый результат: появилось всплывающее окно "Успешно. Операция одобрена Банком", статус в БД APPROVED.

4. Покупка тура при оплате картой с валидным номером 4444 4444 4444 4442 с недостаточным кредитным лимитом.
Ожидаемый результат: появилось всплывающее окно "Ошибка! Банк отказал в проведении операции", статус в БД DECLINED.

#### Негативные сценарии:

Пользователь выбирает покупку с оплатой по дебетовой карте/ или кредитной карте, заполняет невалидными данными поля.

1. Поле "Номер карты".

- Оплата с некорректным номером.
Ожидаемый результат: появилось всплывающее окно об ошибке "Неверный формат".

- Оставить поле пустым.
Ожидаемый результат: появилось сообщение об ошибке "Поле обязательно для заполнения".

- Введен неполный номер карты.
Ожидаемый результат: появилось сообщение об ошибке "Неверный формат".

2. Поле "Месяц".

- Оплата по карте с истекшим сроком действия.
Ожидаемый результат: появилось сообщение об ошибке "Истёк срок действия карты".

- Оставить поле пустым.
Ожидаемый результат: появилось сообщение об ошибке "Поле обязательно для заполнения".

- В поле введены некорректные данные.
Ожидаемый результат: появилось сообщение об ошибке "Неверный формат".

3. Поле "Год".

- Оплата по карте с истекшим сроком действия.
Ожидаемый результат: появилось сообщение об ошибке "Истёк срок действия карты".

- Оставить поле пустым.
Ожидаемый результат: появилось сообщение об ошибке "Поле обязательно для заполнения".

- В поле введены некорректные данные.
Ожидаемый результат: появилось сообщение об ошибке "Неверный формат".

4. Поле "Владелец".

- В поле указаны неполные данные владельца карты, только имя или фамилия.
Ожидаемый результат: появилось сообщение об ошибке "Поле обязательно для заполнения".

- Оставить поле пустым.
Ожидаемый результат: появилось сообщение об ошибке "Поле обязательно для заполнения".

- В поле введены некорректные данные (спецсимволы, цифры).
Ожидаемый результат: появилось сообщение об ошибке "Неверный формат".

5. Поле "CVC/CVV"

- Оставить поле пустым.
Ожидаемый результат: появилось сообщение об ошибке "Поле обязательно для заполнения".

- В поле введены некорректные данные (1 или 2 цифры).
Ожидаемый результат: появилось сообщение об ошибке "Неверный формат".

### Пункт 2. Перечень используемых инструментов.

1. IntelliJ IDEA - среда разработки для приложений на JAVA.
2. JUnit Jupiter - фреймворк для создания автоматизированных тестов..
3. Gradle - система автоматической сборки, позволяющая настраивать зависимости.
4. Selenide - фреймворк для автоматизированного тестирования веб-приложений.
5. Git - система контроля версий. Удобство, возможность одновременной параллельной разработки, есть интеграция с IntelliJ IDEA.
6. MySQL - свободная реляционная система управления базами данных.
7. PostgreSQL - свободная объектно-реляционная система управления базами данных с открытым исходным кодом.
8. Docker - ПО, для автоматизации развёртывания и управления приложениями в средах с поддержкой контейнеризации. Нужен для развертывания виртуальных контейнеров и запуске в них баз данных MySQL и PostgreSQL, а также сервисов банка на NodeJS.
